      Project Thesis Documentation

      Comprehensive overview of problem statements, system architectures, and viva Q&A.

      Problem Statement

      Standardizing student evaluations in modern cohorts suffers from fragmented data streams and delayed warnings. If academic risks are detected only in final exams, corrective coaching is often too late, leading to increased failure rates. This platform addresses this gap by continuously monitoring student attendance, assignments, and engagement to forecast predicted outcomes using explainable Machine Learning.

      Objectives & Methodology

      • Automated Tracking: Aggregate continuous evaluation inputs in LocalStorage databases designed for seamless Firebase/Supabase migrations.
      • Explainable Predictions: Evaluate student risk metrics using Random Forest Decision splits, providing local confidence percentages and dynamic contribution factors.
      • Data Structures Utility: Build adjacency lists for peer relations, max-heaps for class priority queues, and LIFO stacks for grading runs.
      • Actionable Guidance: Generate custom AI-driven tutoring guides and download official printable PDF reports for parents and teachers.
